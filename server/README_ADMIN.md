# WEEZLY Admin Guide

## Доступ к базе данных через Supabase Dashboard

Поскольку WEEZLY использует Supabase (PostgreSQL), управление данными осуществляется через веб-интерфейс Supabase Dashboard.

### Открытие панели управления базой данных

1. **Откройте Supabase Dashboard:**
   - Перейдите по адресу: https://supabase.com/dashboard
   - Войдите в свой аккаунт Supabase

2. **Выберите проект:**
   - В списке проектов найдите ваш проект WEEZLY
   - Нажмите на название проекта для открытия

3. **Откройте Table Editor:**
   - В левом меню выберите **Table Editor**
   - URL будет примерно таким: `https://supabase.com/dashboard/project/YOUR_PROJECT_ID/editor`

### Управление товарами через Table Editor

#### Добавление нового товара

1. В Table Editor выберите таблицу **`products`**
2. Нажмите кнопку **Insert row** (вставить строку)
3. Заполните поля:
   - `name_ru` — название на русском
   - `name_be` — название на беларуском
   - `name_en` — название на английском
   - `desc_ru`, `desc_be`, `desc_en` — описания
   - `price` — цена в USD (например: 999.99)
   - `category` — категория: `laptops`, `smartphones`, `smarthome`, `tablets`, `accessories`
   - `brand` — бренд (например: Apple, Samsung)
   - `stock` — количество на складе
   - `image` — URL изображения
   - `tags` — массив тегов: `{gaming,laptop,powerful}`
   - `specs` — JSON характеристик: `{"CPU": "M2", "RAM": "8GB"}`
   - `mtbf` — среднее время между отказами (для расчета гарантии)
4. Нажмите **Save**

#### Редактирование товара

1. Найдите нужную строку в таблице `products`
2. Нажмите на иконку карандаша (Edit) в конце строки
3. Измените нужные поля
4. Нажмите **Save**

#### Удаление товара

1. Найдите нужную строку в таблице `products`
2. Нажмите на иконку корзины (Delete) в конце строки
3. Подтвердите удаление

### Управление отзывами

#### Просмотр всех отзывов

1. Выберите таблицу **`reviews`**
2. Здесь отображаются все отзывы с полями:
   - `author_name` — имя автора
   - `product_id` — ID товара
   - `rating` — рейтинг (1-5)
   - `comment` — текст отзыва
   - `created_at` — дата создания

#### Удаление фейкового или неуместного отзыва

1. Найдите отзыв в таблице `reviews`
2. Нажмите на иконку корзины (Delete)
3. Подтвердите удаление
4. **Важно:** После удаления автоматически пересчитается средний рейтинг товара

### Просмотр истории заказов

1. Выберите таблицу **`orders`**
2. Здесь отображаются все заказы с полями:
   - `id` — уникальный ID заказа
   - `user_id` — ID пользователя
   - `total` — сумма заказа
   - `status` — статус: `processing`, `paid`, `shipped`, `delivered`
   - `first_name`, `last_name` — имя клиента
   - `email`, `phone` — контакты
   - `address`, `city`, `country` — адрес доставки
   - `created_at` — дата заказа

#### Изменение статуса заказа

1. Найдите заказ в таблице `orders`
2. Нажмите на иконку карандаша (Edit)
3. Измените поле `status` на нужное:
   - `processing` — в обработке
   - `paid` — оплачен
   - `shipped` — отправлен
   - `delivered` — доставлен
4. Нажмите **Save**

#### Просмотр товаров в заказе

1. Выберите таблицу **`order_items`**
2. Каждый товар в заказе отображается отдельно с полями:
   - `order_id` — ID заказа
   - `product_name` — название товара
   - `quantity` — количество
   - `price` — цена за единицу
   - `warranty_level` — уровень гарантии

### Управление пользователями

1. Выберите таблицу **`users`**
2. Здесь отображаются все зарегистрированные пользователи:
   - `id` — уникальный ID
   - `email` — email
   - `name` — имя
   - `role` — роль: `user` или `admin`

#### Назначение администратора

1. Найдите пользователя в таблице `users`
2. Измените поле `role` на `admin`
3. Нажмите **Save**

### SQL Editor для сложных запросов

Для выполнения SQL-запросов:

1. В левом меню выберите **SQL Editor**
2. Нажмите **New query**
3. Введите SQL-запрос, например:

```sql
-- Найти все заказы конкретного пользователя
SELECT * FROM orders WHERE user_id = 'USER_ID_HERE';

-- Найти все отзывы с низким рейтингом
SELECT * FROM reviews WHERE rating <= 2;

-- Средний чек заказов
SELECT AVG(total) FROM orders;

-- Количество заказов по статусам
SELECT status, COUNT(*) FROM orders GROUP BY status;
```

4. Нажмите **Run** для выполнения

### API endpoints для интеграции

Все данные доступны через REST API:

```
GET    /api/products        - список всех товаров
GET    /api/products/:id    - один товар
GET    /api/reviews/:id     - отзывы товара
GET    /api/orders          - заказы пользователя (требуется авторизация)
POST   /api/orders          - создать заказ
```

## Краткая инструкция для демонстрации защиты проекта

### Открытие админ-панели Supabase:
1. Перейдите на https://supabase.com/dashboard
2. Выберите проект WEEZLY
3. Откройте Table Editor

### Что показать:
1. **Товары** — таблица `products` с 16 товарами
2. **Отзывы** — таблица `reviews` (изначально пустая, заполняется пользователями)
3. **Заказы** — таблица `orders` — история всех заказов
4. **Пользователи** — таблица `users` — зарегистрированные пользователи

### Как добавить демо-данные:
1. Откройте SQL Editor
2. Выполните запрос для добавления тестового отзыва:
```sql
INSERT INTO reviews (product_id, user_id, author_name, rating, comment)
VALUES (1, 'test-user-id', 'Тестовый пользователь', 5, 'Отличный ноутбук!');
```

3. Выполните запрос для изменения статуса заказа:
```sql
UPDATE orders SET status = 'shipped' WHERE id = 'ORDER_ID_HERE';
```
